User Management



Application Plantlore allows access to and work with the occurrence data only to the users with a user account created in the application. This way, the data are better protected from damage and misuse. The administrator knows who has access to the occurrence data and who performed any edits to the data. The user manager allows the administrator to add and edit users, set user rights, and also terminate their access to the application. You can open the user manager from menu Data/User management.
User manager dialog


The dialog for user account management is divided into two parts:
  • User list – displays the list of users based on criteria entered for searching and sorting
  • Search – requirements for searching and sorting of users

User list

This part displays the list of users found after a successful search. You can set the number of users displayed in the table yourselves by changing the value for Lines displayed and confirming the new value by the ENTER key. Use Previous and Next buttons to switch over to the previous or next list of users. Information about the currently displayed list of users is stated in the part below the table.
The following buttons enable the work with a record:
  • Add user - adds a user
  • Edit user - edits a user
  • User detail - detailed data and user rights
  • Remove user - abolishes user access

Search

User manager offers the administrator the option to search using the following four parameters: login, name, e-mail, and address. The results may be sorted by login, first name, surname, and date of creation of the user; all of that either in an ascending or descending sequence. Another search criterion is the requirement to display all users, or only users with valid access to the application (the value of item Date of user removal is empty for these users). If no parameters and limitations are entered, all users will be found.

Note: the search is case sensitive.

Adding a user

Add user dialog


Open the Add user dialog from User manager. The window displayed offers the following items:

User data

  • Login - user login that will be used for access to the application. This item is obligatory and you will not be able to add a user without filling it in. The login must be unequivocal in the database and it will not be editable after you save the user.
  • Password - Plantlore user password. This item is obligatory.
  • Name - first name of the user. This item is obligatory.
  • Surname - surname of the user. This item is obligatory.
  • E-mail - user’s e-mail address. This item is obligatory.
  • Address - user’s address. This item is obligatory.
  • Note - administrator’s note about the user.

User rights

  • Administrator right – by selecting it, the user will be granted same rights as the administrator. You should carefully consider whom to assign administrator rights. Recommendation: in order to provide for higher security, there should only be one user with administrator rights.
  • Right to edit all – by selecting it, the user is granted the right to edit all occurrences in the database.
  • Right to add records – by selecting it, the user is granted the right to add new occurrences.
  • Right to edit other users’ occurrences – this item contains the list of users whose occurrences the user will be permitted to edit.

Editing a user

Edit user dialog


Open the Edit user dialog from User manager. Don’t forget that you must select a user to open this dialog. The layout of the Edit user dialog is almost identical with the Add user dialog. The obligatory items are: password, name, surname, e-mail.
Non-editable items::
  • Login
  • Date of creation: date and time of user creation. Non-editable item.
  • Date of removal: date and time of abolishment of user access to the application. Non-editable item.

Detailed user data and rights

You can open the dialog displaying detailed information about the user and his rights from User manager. You have to select a user in the list of users to be able to open this dialog. The dialog is almost identical with the Edit user dialog. All items are non-editable, their nature is informative.

Removing user from the system

In order to abolish the user access to the application, use button Remove user in the User manager. Before pressing this button, it is necessary to select a user in the list of users. After the button is pressed, the user will be informed about the operation about to be performed. By confirming the operation, the current time will be stored in the item Date of user removal.



Renewal of access for a user that has been removed means adding a new user. Login of a new user must be unequivocal in the database (i.e. it is not possible to use a login of user who has or had an account in a given database). This way, the database stores the information about the periods during which a user had access to the application.

User rights

The administrator sets rights for work with the occurrence data while adding or editing a user.
Setting the rights


User can have the following rights:
  • Right to browse the data – this right does not have to be set anywhere as it is granted automatically with the creation of access to the application
  • Right to add data – this right permits the user to add occurrences, authors, and publications. He is also automatically granted the right to edit the records that he added himself and to browse all the occurrences. Moreover, he has the right to display the history of the occurrence that he added himself.
  • Right to edit other users’ data – a user (Joe) may ask the administrator to add a specific user (Frank) to the list of users whose records he may edit (this right to edit implies the right to access the record history). Whether a user is added into this list depends on the administrator and/or mutual agreement of the users.
  • Administrator right – a user with administrator right has, of course, the largest collection of rights – he may edit the occurrences, metadata, publications, authors, plants, and users. He has the access to the history of individual occurrences, but also to the whole history.
